Azure-erőforrások üzembe helyezése és kezelése közvetlenül a szerkesztőből

Hatékony szerkesztés és hibakeresés

Az IntelliSense használatával jelentősen növelheti az eredményességet: az intelligens kódkiegészítés a változótípusokat, a függvénydeklarációkat és az importált modulokat is felhasználja. Nagy mennyiségű kódban is egyszerűen navigálhat az olyan funkciókkal mint a Go To Definiton (Definícióra ugrás) vagy a Find All References (Összes hivatkozás keresése). A hibakeresést közvetlenül a kódszerkesztőben végezheti el. Ehhez töréspontokat, egy teljes hívássorozatot és egy interaktív konzolt használhat.

Munkáját a JavaScript és a Node.js használatával teheti egyszerűbbé

A JavaScripthez használható automatikus importálás és a TypeScript használatával automatikusan adhat hozzá importált utasításokat helyi és NMP-függőségekből. Még egyszerűbb JavaScript esetén is kihasználhatja a TypeScript típusellenőrzési funkcióját.

Kiszolgáló nélküli megoldások fejlesztése és hibakeresése a helyszínen

Időt és erőforrásokat takaríthat meg azzal, hogy az eseményvezérelt alkalmazásokat helyileg futtathatja és tesztelheti pontosan ugyanazzal a futtatókörnyezettel, amelyet az Azure Functions is használ a felhőben, majd pedig közvetlenül a szerkesztőből végezheti el az üzembe helyezést, ha elkészült az alkalmazás.

Több száz nyelv támogatása

A Visual Studio Code szinte minden fontosabb programozási nyelvet támogat. A csomagban beépítetten elérhetőek a leggyakrabban használt webalapú nyelvek, többek között a JavaScript, a TypeScript, a CSS és a HTML, és emellett a Visual Studio piacteréről származó több száz bővítmény használható kódkiegészítéshez, lint-analízishez, hibakereséshez, újrabontáshoz és sok más funkcióhoz.

Nyelvi kiterjesztések megjelenítése

Kapcsolódás fejlesztői szolgáltatásokhoz és erőforrásokhoz

A Visual Studio Code-dal egyszerűen dolgozhat párhuzamosan helyi és távoli erőforrásokkal. Helyszíni MongoDB-adatbázist használó, tárolóalapú alkalmazásokat is fejleszthet és tesztelhet vele. Az elkészült tárolót üzembe helyezheti az App Service-ben, az adatbázist pedig feltöltheti a Cosmos DB-be – mindezt ráadásul a kódszerkesztő elhagyása nélkül.
